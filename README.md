# üçû BabyBread

A web app for choosing a baby's name by having friends and family vote on head-to-head duels between candidate names. Rankings are computed using the [ELO rating system](https://en.wikipedia.org/wiki/Elo_rating_system), the same algorithm used in chess.

**Live:** https://babyname-e092d.web.app

---

## What it does

- Anyone with a Google account (or email/password) can join and suggest names for both genders
- Names are presented in one-on-one duels: pick the one you like more
- Each vote updates ELO scores ‚Äî names that beat strong competition rank higher than names that only beat weak ones
- A global ranking shows the community's consensus; a personal ranking shows each voter's own preferences
- Every voter's ranking is shown independently; the global view averages them all
- Admins can manage users, delete votes, and reset the database

## How ELO works here

Every name starts at 1000 points. When you vote A over B, points transfer from B to A ‚Äî the exact amount depends on the rating gap (a surprise upset transfers more than an expected win). K-factor is 32.

Scores are stored per user, so deleting a vote triggers a full replay of that user's remaining match history to recompute accurate ELO from scratch.
